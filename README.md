# Suru Chatbot — Phase 1 (Login + Home Feed)

এই ধাপে যা বানানো হয়েছে:
- **index.html** — লগইন: ফোন নাম্বার → নাম → OTP (Firebase Phone Auth)। একবার লগইন করলে সরাসরি home.html-এ চলে যায়, দ্বিতীয়বার লগইন চাইবে না।
- **home.html** — নিউজফিড: প্রোফাইল স্ট্রিপ, পোস্ট করার বক্স (ছবি/ভিডিও সহ), ফোন নাম্বার দিয়ে ফ্রেন্ড সার্চ, পোস্ট কার্ড (৫টি রিয়্যাকশন long-press করে, কমেন্ট + রিপ্লাই + কমেন্ট লাইক, সেভ, শেয়ার — WhatsApp/imo সহ), পোস্টের ৩-ডট মেনু (নিজের পোস্ট এডিট/ডিলিট, অন্যের পোস্ট হাইড/ব্লক), সাইড মেনু (সেটিং, লগ-আউট, সেভড, Dark/Light থিম)।
- ভিডিও ট্যাব, চ্যাট, প্রোফাইল পেজ, নোটিফিকেশন — পরের ধাপগুলোয় যোগ হবে (এখন বাটনে চাপ দিলে "শীঘ্রই আসছে" টোস্ট দেখাবে)।

## ১. Firebase Console-এ যা চালু করতে হবে (একবারই)

1. **Authentication → Sign-in method → Phone** চালু করুন।
2. **Authentication → Settings → Authorized domains**-এ আপনার GitHub Pages ডোমেইন (যেমন `yourname.github.io`) যোগ করুন — নাহলে OTP পাঠাবে না।
3. **Firestore Database** তৈরি করুন (production mode)।
4. **Storage** চালু করুন।
5. `firestore.rules` ফাইলের কনটেন্ট Firestore → Rules ট্যাবে পেস্ট করে Publish করুন।
6. `storage.rules` ফাইলের কনটেন্ট Storage → Rules ট্যাবে পেস্ট করে Publish করুন।

> টেস্ট করার সময় ফোন নাম্বার আসল সিম না হলে, Authentication → Sign-in method → Phone-এর নিচে **Phone numbers for testing** যোগ করে একটা fixed OTP (যেমন 123456) সেট করে দ্রুত টেস্ট করতে পারেন।

## ২. রান করা

`js/firebase-config.js`-এ আপনার নিজের Firebase config বসানো আছে। কিছু বদলানোর দরকার নেই যদি না আপনি নতুন প্রজেক্ট বানান।

লোকালি টেস্ট করতে যেকোনো static server লাগবে (`file://` দিয়ে Firebase phone auth কাজ করবে না):
```
npx serve .
```

## ৩. GitHub-এ হোস্ট করা (GitHub Pages)

1. এই পুরো ফোল্ডারটা একটা নতুন GitHub রিপোতে push করুন।
2. রিপোর **Settings → Pages → Branch: main / (root)** সিলেক্ট করে Save করুন।
3. কিছুক্ষণ পর `https://<username>.github.io/<repo-name>/` লিংকে অ্যাপ চালু হবে।
4. ওই লিংকটাই Firebase-এর Authorized domains-এ যোগ করতে ভুলবেন না (ধাপ ১.২)।

## ৪. AppMint (বা যেকোনো website-to-app wrapper) দিয়ে অ্যাপে রূপান্তর

- Wrapper-এ GitHub Pages লিংকটা দিন।
- আগে OTP স্ক্রিন না আসার যে সমস্যা হয়েছিল, তার সবচেয়ে বড় কারণ সাধারণত reCAPTCHA WebView-তে ঠিকমতো লোড না হওয়া। তাই wrapper-এর সেটিংসে **JavaScript enabled** এবং **third-party cookies / popups allowed** চালু আছে কিনা দেখে নিন। সমস্যা থাকলে amI জানাবেন, আলাদাভাবে ডিবাগ করব।

## পরের ধাপ

আপনি বললেই এর পরে **চ্যাট সিস্টেম** (আগের কোডের উপর আপডেট) নিয়ে শুরু করব — চ্যাট লিস্ট, ১:১ ও গ্রুপ মেসেজিং, সেন্ট/সিন স্ট্যাটাস, ব্লক/মিউট।

# Phase 2 — Chat System (নতুন যোগ হলো)

নতুন পেজ:
- **chatlist.html** — স্টোরি বার (24h), ফ্রেন্ড রিকোয়েস্ট ব্যানার, ফোন নাম্বার দিয়ে সার্চ, চ্যাট লিস্ট (কল/ভিডিও-কল আইকন, ৩-ডট মেনু: nickname/mute/block/delete)
- **requests.html** — ইনকামিং ফ্রেন্ড রিকোয়েস্ট Accept/Decline (Accept করলেই দুজনের friends লিস্টে যোগ হয় এবং একটা চ্যাট থ্রেড তৈরি হয়)
- **chat.html?chatId=...** — মেসেজ উইন্ডো: টেক্সট, ছবি/ভিডিও, ভয়েস মেসেজ (রেকর্ড করে পাঠানো), Sent/Seen স্ট্যাটাস, মেসেজে long-press করে ৫টি রিয়্যাকশন, উপরে নাম ট্যাপ করলে contact/group info sheet (nickname, mute, share contact, group তৈরি, block, delete chat, group হলে member list)
- **creategroup.html** — friends লিস্ট থেকে সর্বোচ্চ ১০০ জন বেছে গ্রুপ নাম ও ছবি দিয়ে গ্রুপ তৈরি
- **js/common.js** — এই পেজগুলোর শেয়ার্ড helper ফাংশন

home.html-এর 💬 আইকন এখন সরাসরি chatlist.html-এ নিয়ে যায় এবং পেন্ডিং রিকোয়েস্ট থাকলে ছোট লাল ডট দেখায়।

`firestore.rules` ও `storage.rules` আপডেট হয়েছে — Firestore ও Storage-এর Rules ট্যাবে নতুন কনটেন্টটা আবার পেস্ট করে Publish করুন।

## নতুন যা করতে হবে Firebase Console-এ

1. **Firestore composite index:** চ্যাট লিস্ট লোড করার সময় (`members array-contains` + `orderBy lastMessageAt`) একটা কম্পোজিট ইনডেক্স লাগবে। প্রথমবার chatlist.html খুললে browser console-এ Firebase একটা লিংক দেবে ("The query requires an index... click here") — সেই লিংকে ক্লিক করলেই ১ ক্লিকে ইনডেক্স তৈরি হয়ে যাবে। কয়েক মিনিট wait করতে হতে পারে।
2. ভয়েস মেসেজের জন্য ব্রাউজার মাইক্রোফোন পারমিশন চাইবে — অনুমতি দিতে হবে।

## এখনো বাকি (পরের ধাপে আসবে)

- অডিও/ভিডিও কল (ZegoCloud ইন্টিগ্রেশন) — এখন কল বাটনে চাপ দিলে শুধু "শীঘ্রই আসছে" দেখাবে

# Phase 3 — Suru ভিডিও + পুশ নোটিফিকেশন/কুইক-রিপ্লাই

## Suru ভিডিও (video.html)
TikTok-স্টাইল ভার্টিক্যাল সোয়াইপ ফিড। স্ক্রিনে যেই ভিডিওটা দেখা যাচ্ছে শুধু সেটাই অটোপ্লে হয় (বাকিগুলো pause, তাই র‍্যাম/ব্যান্ডউইথ বাঁচে = দ্রুত লোড)। প্রতি ভিডিওতে: পোস্টকারীর নাম + Follow বাটন (নামে ট্যাপ করলে তার সব ভিডিও — `video.html?uid=...`), হার্ট লাইক, কমেন্ট শীট, শেয়ার, টপিক দিয়ে সার্চ (`#topic`), এবং ➕ বাটনে ভিডিও আপলোড (ক্যাপশন + টপিক সহ, আপলোড প্রোগ্রেস বার দেখায়)।

> **সততার সাথে একটা কথা বলে রাখি:** "ফোন মিনিমাইজ করলেও ভিডিও চলতে থাকা" — এটা সাধারণ ব্রাউজার/ওয়েবভিউ (AppMint-এর মতো wrapper) দিয়ে সম্ভব না; এটা করতে হলে একদম নেটিভ Android অ্যাপ (Kotlin/Java বা React Native) লাগবে background media session নিয়ে। এখন যা আছে তাতে ভিডিও ট্যাবে থাকা অবস্থায় স্মুথলি চলবে, কিন্তু অ্যাপ মিনিমাইজ/স্ক্রিন লক করলে থেমে যাবে।

## পুশ নোটিফিকেশন + কুইক-রিপ্লাই

নতুন ফাইল:
- **firebase-messaging-sw.js** (root-এ, index.html-এর পাশে থাকতে হবে) — অ্যাপ বন্ধ থাকা অবস্থায় নোটিফিকেশন দেখায়
- **js/push.js** — পারমিশন চাওয়া, টোকেন সেভ করা, ফোরগ্রাউন্ডে টোস্ট দেখানো
- **functions/** — Cloud Function যেটা নতুন মেসেজ এলে বাকি মেম্বারদের push পাঠায় (mute থাকলে পাঠায় না)
- **firebase.json** — deploy করার কনফিগ

### সেটআপ (৩টা ধাপ, একবারই করতে হবে)

**১. VAPID key জেনারেট করুন**
Firebase Console → Project settings → Cloud Messaging → Web configuration → Generate key pair। যেই key টা পাবেন সেটা `js/push.js`-এর `FCM_VAPID_KEY = ""` এর মধ্যে বসিয়ে দিন।

**২. Cloud Function ডিপ্লয় করুন** (নতুন মেসেজ এলে push পাঠানোর জন্য এটা লাগবেই)
```
npm install -g firebase-tools
firebase login
firebase use --add        # আপনার chatbot-c1a41 প্রজেক্ট সিলেক্ট করুন
cd functions && npm install && cd ..
firebase deploy --only functions
```
এর জন্য Firebase-এ **Blaze (pay-as-you-go)** প্ল্যানে যেতে হবে — প্রথম কয়েক লাখ নোটিফিকেশন পর্যন্ত ফ্রি টায়ারেই থাকবে, বিল আসার কথা না একটা ছোট অ্যাপের জন্য।

**৩. GitHub Pages-এ push করার আগে** `firebase-messaging-sw.js` ফাইলটা রুট ফোল্ডারেই আছে কিনা নিশ্চিত করুন (এটা সরানো/নাম বদলানো যাবে না)।

### "উপর থেকে রিপ্লাই" আসলে কীভাবে কাজ করবে

আপনি চেয়েছিলেন নোটিফিকেশন থেকেই অ্যাপে না ঢুকে রিপ্লাই দেওয়া (Android-এর নেটিভ apps যেমন WhatsApp-এ হয়)। এটার জন্য দুইটা লেভেল রেখেছি:

1. **সবজায়গায় কাজ করবে (মূল সিস্টেম):** নোটিফিকেশনে ট্যাপ করলেই সরাসরি সেই চ্যাটে ঢুকে যাবে আর টেক্সট বক্সে কার্সার অটো-ফোকাস হয়ে থাকবে — মানে ১টা ট্যাপ, তারপর সাথে সাথে টাইপ করা যাবে। Firestore ক্যাশিং করা আছে বলে এটা প্রায় সাথে সাথে খুলবে।
2. **যেখানে ব্রাউজার সাপোর্ট করে (বোনাস, mainly Chrome desktop):** নোটিফিকেশনেই একটা "Reply" টেক্সট বক্স দেখাবে, লিখে সেন্ড করলে অ্যাপ না খুলেই মেসেজ চলে যাবে।

> **সততার সাথে আরেকটা কথা:** Android-এ Chrome/ওয়েব বা AppMint-এর মতো webview wrapper থেকে **আসলেই notification shade-এর ভেতরে টাইপ করে সেন্ড করা** (যেমনটা WhatsApp নেটিভ অ্যাপে হয়) — এটা ব্রাউজারের Web Notification API দিয়ে রিলায়েবলি সম্ভব না; এটা native Android app-এর একটা বিশেষ ফিচার (RemoteInput)। আমি সেই ফিচারটার সবচেয়ে কাছের ও দ্রুততম বিকল্প বানিয়ে দিয়েছি (ট্যাপ → সরাসরি টাইপিং-এ), যেটা বাস্তবে খুব দ্রুত লাগবে। সত্যিকারের in-shade typing চাইলে ভবিষ্যতে পুরো অ্যাপটা নেটিভ Android (Kotlin) এ রিরাইট করতে হবে — সেটা একটা আলাদা বড় কাজ, চাইলে পরে সেটা নিয়েও কথা বলতে পারি।

### পারফরম্যান্স (দ্রুত লোডের জন্য যা করা হয়েছে)
- Firestore offline persistence (আগেই করা) — পুরনো ডেটা সাথে সাথে দেখায়, নতুনটা ব্যাকগ্রাউন্ডে সিঙ্ক হয়
- প্রতিটা লিস্ট কোয়েরিতে limit (ফিড ৪০, চ্যাট ৬০, মেসেজ সর্বশেষ ৫০, ভিডিও ২০)
- ভিডিও ফিডে শুধু স্ক্রিনে থাকা ভিডিওটাই play হয়, বাকিগুলো pause — ব্যান্ডউইথ ও ব্যাটারি বাঁচে
- push notification data-only payload — extra HTTP round-trip ছাড়াই সরাসরি সার্ভিস ওয়ার্কার/অ্যাপ রেন্ডার করে

# Phase 4 — অডিও/ভিডিও কল (ZegoCloud)

নতুন ফাইল:
- **call.html + js/call.js** — আসল কল রুম, ZegoCloud-এর Prebuilt UIKit ব্যবহার করে (একটাই CDN স্ক্রিপ্ট দিয়ে সম্পূর্ণ কল UI — ক্যামেরা/মাইক টগল, স্ক্রিন শেয়ার, লেআউট — সব রেডি)
- **js/callSignal.js** — পুরো অ্যাপ জুড়ে (home, chat list, chat, video — যেখানেই থাকুন) ইনকামিং কল লিসেন করে; কেউ কল করলে সাথে সাথে Accept/Decline স্ক্রিন দেখাবে
- **js/zego-config.js** — আপনার ZegoCloud AppID + AppSign (আগে থেকেই বসানো আছে)

## কীভাবে কাজ করে
1. চ্যাট হেডারের 📞/🎥 বা চ্যাট-লিস্টের কল আইকনে চাপ দিলে একটা `calls` ডকুমেন্ট তৈরি হয় (status: "ringing") এবং কলার সরাসরি call.html-এ চলে যায়
2. যাকে কল করা হয়েছে, সে অ্যাপের যেকোনো পেজে থাকুক না কেন — অটোমেটিক Accept/Decline স্ক্রিন দেখবে
3. Accept করলে দুজনেই একই রুমে (roomID = chatId) জয়েন হয়ে যায় — অডিও কলে ক্যামেরা বন্ধ থাকে, ভিডিও কলে চালু
4. গ্রুপ চ্যাটেও একই সিস্টেম কাজ করে — roomID গ্রুপের chatId, তাই একাধিক মেম্বার একসাথে জয়েন করতে পারবে (group call)
5. কেউ ৪০ সেকেন্ডে রিসিভ না করলে "missed call" হয়ে যাবে; Decline করলে কলার সাথে সাথে জানতে পারবে

## জরুরি — প্রোডাকশনে যাওয়ার আগে একটা কথা
এখন `js/zego-config.js`-এ থাকা **ServerSecret ব্রাউজারেই আছে** (দ্রুত টেস্ট করার জন্য `generateKitTokenForTest` ব্যবহার করা হয়েছে)। এটা ডেভেলপমেন্ট/অল্প কিছু ইউজারের জন্য ঠিক আছে, কিন্তু বড় পরিসরে পাবলিক লঞ্চ করার আগে টোকেন জেনারেশনটা একটা ছোট Cloud Function-এ সরিয়ে নেওয়া উচিত (secret কখনো ক্লায়েন্টে থাকা উচিত না) — চাইলে সেটাও পরে বানিয়ে দিতে পারি।

## Firebase Console-এ নতুন কিছু লাগবে কিনা
`calls` কালেকশনের কোয়েরির জন্য (`members array-contains` + `status ==`) সম্ভবত আরেকটা composite index চাইবে — chatlist/chat/home প্রথমবার খোলার সময় console-এ লিংক আসলে ক্লিক করে বানিয়ে নিন (আগের ইনডেক্সের মতোই)।
